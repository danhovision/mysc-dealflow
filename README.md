# 🔍 MYSC 딜플로우 인텔리전스 에이전트

> 투자 검토 요청 이메일을 자동으로 분석하고 Slack으로 브리핑하는 AI 에이전트

## 개요

MYSC 투자심사역의 딜 소싱 업무를 자동화하는 에이전트입니다.
Gmail로 수신되는 투자 검토 요청 이메일을 AI가 자동 분석하여
매일 Slack으로 브리핑을 발송합니다.

## 주요 기능

- 📥 Gmail 자동 모니터링 (읽지 않은 IR 이메일 수집)
- 🤖 Claude AI 기반 16개 항목 자동 추출
- 📁 A/C/H 폴더 자동 분류
  - 🟢 A폴더: 현재 최적화 (검증된 시장, 경쟁 우위)
  - 🔵 C폴더: 미래 최적화 (멱함수 잠재력, 비대칭 수익 가능)
  - 🟤 H폴더: 지역의 숨은 히어로 (지역 강소기업)
- 💬 Slack 팀 브리핑 자동 발송
- 🔗 7개 액셀러레이팅 프로그램 연계 추천
- 💚 소셜벤처 판별

## 분석 항목 (16개)

기업명, 대표자, 법인소재지, 창업연한, 투자단계, 희망투자금액,
산업분류, 재직자수, 매출, 특허, 벤처기업인증, 창업확인서,
TIPS, LIPS, 소셜벤처판별, 최근 뉴스

## 개발

MYSC 에이블(김정태) × Claude (Anthropic)


1. 에이전트 목적 및 배경
1.1 해결하고자 하는 문제
투자 심사역은 매일 다수의 투자 검토 요청 이메일을 수신하나, 개별 이메일에 대한 1차 분석과 회신에 소요되는 시간이 과도하다. 이로 인해 실질적인 가치가 있는 딜을 놓칠 리스크와 함께, 발송자에 대한 적시 응대가 지연되는 문제가 발생한다.

핵심 문제 3가지
① 수신 이메일 증가 → 1차 검토에 과도한 시간 소모
② 분류 기준 미적용 → 중요 딜(특히 C폴더·H폴더)을 놓칠 리스크 상존
③ 회신 지연 → 발송자 경험 저하 및 네트워크 관계 훼손

1.2 에이전트 한 줄 정의
투자 검토 요청 이메일을 자동으로 수집·분석·분류(A/C/H 폴더)하고, 펀드 정합성·액셀러레이팅 연계 가능성까지 판별하여 심사역 전원에게 일일 통합 브리핑을 제공하며, 유형별 맞춤 회신을 자동 발송하는 딜플로우 인텔리전스 에이전트

1.3 에이전트가 대체하는 것 vs 증폭하는 것
대체 (Automate)	증폭 (Augment)
이메일 1차 열람 및 기본 정보 추출	심사역의 최종 투자 판단
공개 데이터 검색 (뉴스·DART·벤처인)	A/C/H 폴더 AI 판단의 최종 확인
A/C/H 폴더 1차 분류	캐주얼 IR 참석 여부 결정
펀드 정합성 자동 매칭	관계 맥락(네트워크 연결고리) 판단
액셀러레이팅 프로그램 연계 추천	3유형 추가 질문 내용 선택 및 커스터마이징
유형별 표준 답신 초안 생성	팀 내 딜 배분 및 우선순위 최종 결정

2. 전체 워크플로우 설계
2.1 플로우 개요
5단계 자동화 파이프라인 (v2.0)
STEP 1  |  [트리거]  Gmail 수신 감지 → 투자 검토 요청 이메일 판별 (개인별 + 팀 통합)
STEP 2  |  [분석]    이메일 본문 + 첨부파일 파싱 → 16개 항목 추출 + 웹검색 보강
STEP 3  |  [분류]    A/C/H 폴더 판정 + 펀드 정합성 + 액셀러레이팅 연계 추천
STEP 4  |  [브리핑]  매일 오전 9시 Slack → 개인 채널(본인 수신) + 팀 통합 채널(전체)
STEP 5  |  [회신]    심사역 유형 선택(1·2·3) → 맞춤 이메일 자동 발송 + Airtable 로그

2.2 STEP 1 — 이메일 수집 및 판별
판별 기준	세부 내용
포함 키워드	IR, 투자 검토, 투자 요청, 사업계획서, 데크, 피칭, 창업팀, 시리즈, 라운드, 소셜벤처 등
첨부 파일	PDF·PPT·PPTX 포함 여부 (IR 덱 가능성)
멀티 심사역	팀 내 심사역 전원의 Gmail 동시 모니터링, 중복 수신 자동 감지
제외 대상	뉴스레터, 자동발송 알림, 이미 처리된 스레드 답장

2.3 STEP 2 — 자동 분석 항목 (16개) + 첨부파일 처리
카테고리	분석 항목	소스
기업 기본	기업명 / 법인 소재지 / 창업 연한	이메일 + DART
투자 정보	투자 단계 / 라운드 정보 / 희망 투자금액	이메일
산업 분류	산업 카테고리 (MYSC 분류 기준 매핑)	이메일 + AI 분류
기업 현황	재직자 수 / 매출 규모 / 특허 보유 현황	이메일 + DART
인증·자격	벤처기업 인증 / 창업확인서 / 소셜벤처 판별	이메일 + 공개DB
정책 연계	TIPS 선정 여부 / LIPS 해당 여부	이메일 + 공개DB
외부 정보	최근 보도자료·기사 최대 5건	웹검색

첨부파일 처리 파이프라인
① PDF·PPT·PPTX → 텍스트 레이어 추출 시도 (PyMuPDF)
② 텍스트 추출 실패 또는 텍스트량 부족 → OCR Tool 호출 (Google Vision / AWS Textract)
③ OCR 결과도 신뢰도 낮음 (복잡한 표·차트·한글 폰트 깨짐) → 브리핑에 '📎 수동 확인 필요' 플래그
④ HWP → LibreOffice 변환 후 텍스트 추출
⑤ Excel·CSV → 수치 데이터 직접 파싱 (재무제표·매출 데이터)
※ 본문과 첨부 간 정보 충돌 시: 첨부파일 정보 우선 적용

2.4 STEP 4 — Slack 브리핑 구조 (개인 + 팀 통합)
채널 유형	내용
개인 채널 (심사역별)	본인이 수신한 딜만 표시 / 회신 유형 선택 버튼 포함 / 본인만 볼 수 있음
팀 통합 채널 (전체 공유)	팀 전체 수신 딜 통합 표시 / 담당 심사역 표시 / 중복 수신 건 병합 처리
중복 수신 처리	동일 기업이 복수 심사역에게 발송 시 → 먼저 수신한 심사역이 주담당 / 팀 채널에 중복 알림

팀 통합 Slack 브리핑 포맷 (예시)

🟢 A폴더 2건  |  🔵 C폴더 3건  |  🟤 H폴더 1건  |  ⚪ 판단불가 1건
────────────────────────────────────────────────────
① [C폴더] 코스모스랩류 수계배터리 스타트업 | Pre-A | 에너지 | 담당: 에이블
   시그널: 역접근 + 시장선행 / 신뢰도: 보통
   최근 기사: '차세대 배터리 스타트업 산업부 과제 선정' (1.13)
② [H폴더] 경북 안동 전통장류 제조업체 | — | 식품제조 | 담당: OOO

3. 딜 분류 체계 — A / C / H 폴더
3.1 3폴더 분류 개요
모든 딜은 아래 세 가지 폴더 중 하나로 분류된다. AI가 1차 판정하고, 심사역이 브리핑에서 최종 확인한다.

폴더	별칭	핵심 논리	대표 사례
🟢 A폴더	현재 최적화	검증된 시장, 경쟁 속 우위가 있는 팀	—
🔵 C폴더	미래 최적화	지금은 안 보이지만 멱함수 잠재력을 가진 팀	식스티헤르츠, 트래쉬버스터즈, 코스모스랩
🟤 H폴더	지역의 숨은 히어로	이미 견고하지만 VC 레이더에 안 잡힌 지역 강소기업	디에스푸즈, 시그널케어, 소담소담

3.2 A폴더 — 현재 최적화
A	A폴더 — 현재 최적화형
시장이 이미 존재하고, 경쟁자도 있지만 그 안에서 더 빠르고 더 잘하는 팀. 투자 논리가 명확하고 레퍼런스가 있어 심사위원회 설득이 용이한 딜.
•	동일 문제 해결 경쟁사 3개 이상 존재하는 시장에서 활동
•	국내외 유사 BM 성공 사례가 명확히 존재
•	현재 시장 수요가 검증되어 있음
•	스케일업 경로가 업계 표준과 유사
📌 앵커 사례: —

3.3 C폴더 — 미래 최적화
C	C폴더 — 미래 최적화형
현재는 시장이 잘 보이지 않고 성공 가능성에 의심이 드는 팀. 하지만 멱함수 법칙에 따라 초기 VC가 놓칠 수밖에 없는, 향후 비대칭적 거대 수익이 가능한 잠재팀. 아래 시그널 중 2개 이상 해당 시 C폴더 판정.
•	시그널 1 | 역접근 (Counter-approach): 지배적 기술·방식과 구조적으로 다른 접근 → 코스모스랩 (수계배터리로 화재 위험 자체를 제거)
•	시그널 2 | 족보 없음 (No Precedent): 국내외 유사 BM 선례 없거나 매우 희소 → 트래쉬버스터즈 (없던 다회용컵 B2B 시장을 직접 창출)
•	시그널 3 | 시장 선행 (Market Timing): 현재 수요 낮지만 구조적 변화로 수요 예측 → 식스티헤르츠 (재생에너지 간헐성), 스튜디오갈릴레이 (인구감소 지역 수요맞춤 모빌리티)
•	시그널 4 | 문제정의 창의성: 업계 관점과 다른 프레임, 특히 '약점을 자산으로 뒤집는 구조' → 코액터스 (청각장애인 드라이버), 해녀의부엌 (소멸 위기 해녀 문화 × 파인다이닝)
•	시그널 5 | 임팩트 내재형: 사회적 가치 실현이 BM 구조 자체에 내재화 → 에이아이웍스 (발달장애인 고용이 AI 생산 핵심), 수퍼빈 (폐플라스틱 회수가 곧 원료 수급)
📌 앵커 사례: 코스모스랩 · 트래쉬버스터즈 · 식스티헤르츠 · 스튜디오갈릴레이 · 코액터스 · 해녀의부엌 · 에이아이웍스 · 수퍼빈

3.4 H폴더 — 지역의 숨은 히어로
H	H폴더 — 지역의 숨은 히어로
매출·이익이 이미 견고하지만 스케일업 스토리가 약해 기존 VC 레이더에 안 잡혀온 지역 강소기업. 지방시대 펀드 흐름에서 재조명될 잠재력 보유. 아래 시그널 중 3개 이상 해당 시 H폴더 판정.
•	시그널 1 | 재무 견고성: 매출·이익이 이미 안정적으로 발생
•	시그널 2 | 지역 산업 연계: 제조업·농업·수산업 등 지역 주력산업과 깊이 결합 → 시그널케어 (청도 그린바이오), 디에스푸즈 (영주 사과)
•	시그널 3 | 지역 고유성: 해당 지역 아니면 불가능한 원료·문화·인프라가 핵심 자산 → 디에스푸즈 (영주 고유 사과 품종)
•	시그널 4 | 정책 수혜 예정: 지방시대 펀드·지역재생 뉴딜 등 정책 자금 유입 예상
•	시그널 5 | VC 저평가: 스케일업 스토리 약해 기존 VC가 지나쳤을 가능성 → 소담소담 (부산 육가공 제조업)
📌 앵커 사례: 디에스푸즈 (영주) · 시그널케어 (청도) · 소담소담 (부산)

3.5 AI 판단 프롬프트 구조 (에이전트 삽입용)
A/C/H 폴더 판단 지시문
아래 기준으로 이 팀을 A / C / H 폴더 중 하나로 분류하라.
복수 해당 시 주된 특성 기준으로 하나만 선택하되, 보조 특성은 별도 메모로 남겨라.

[A폴더] 동일 문제 경쟁사 3개 이상 / 유사 BM 성공 사례 명확 / 현재 수요 검증

[C폴더] 아래 시그널 중 2개 이상 해당:
  1. 역접근: 지배적 기술·방식과 구조적으로 다른 접근
  2. 족보 없음: 국내외 유사 BM 선례 없거나 매우 희소
  3. 시장 선행: 현재 수요 낮지만 구조적 변화로 수요 예측 (재생에너지, 인구감소 등)
  4. 문제정의 창의성: 업계 관점과 다른 프레임 / 약점을 자산으로 뒤집는 구조
  5. 임팩트 내재형: 사회적 가치가 BM 구조 자체에 내재화

[H폴더] 아래 시그널 중 3개 이상 해당:
